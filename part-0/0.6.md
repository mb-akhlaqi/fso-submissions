## Exercise 0.6: New note in Single page app diagram
### Notable events that transpire between browser and server:

1. User types in a new note and submits (HTML form).
2. JS code intercepts the submission (preventDefault).
3. JS code creates a note object.
4. JS code adds the note to the local `notes` array.
5. JS code re-renders the notes list.
6. Browser sends a `POST` request to `/new_note_spa` containing new note data.
7. Server stores the new note.
8. Server sends a response with `201 Created` status code.

Note: Also, even if the `POST` request is unsuccessful the browser still shows the new note as added.
