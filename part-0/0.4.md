## Exercise 0.4: New Note Diagram
### Notable events that transpire between browser and server:

1. User types and submits a new note.
2. Browser sends a `POST` request to `/new_note` containing the new note data.
3. Server stores the new note.
4. Server send a response with `302 Found` status code (redirect).
5. Browser redirects to `/notes` (a refresh).
6. Server returns the HTML document (probably `index.html`).
7. Browser requests `main.css`.
8. Broswer requests `main.js`.
9. Browser executes the JS code.
10. Browser requests `data.json`.
11. Server returns the updated list of notes.
12. Browser renderes the notes.
